# lifeup_mod_coins

Obsidian 插件（草稿版）：把 Markdown 待办同步到 LifeUp（通过云人升 API）。

## 功能概览

- 右键同步：当前待办 / 选中待办
- 日记同步：手动按钮（推荐）或自动模式
- 子任务同步：支持一层 subtask；多级会拍平
- 数据面板：金币、番茄、指定物品、`query_skill`（1~6）

---

## 一次性准备（手机端）

1. 安装 LifeUp：<https://lifeupapp.fun/zh/index.html#cta>
2. 在 LifeUp 打开：侧边栏 → 设置 → 实验 → 开发者模式
  - 开启后可在任务/商品/清单详情看到 ID
3. 安装并启动《云人升》
  - 国内会员版可在 LifeUp 实验页下载
  - 或 Google Play：<https://play.google.com/store/apps/details?id=net.lifeupapp.lifeup.http>
4. 在《云人升》里授权并启动服务，记录 `IP:端口`
  - 示例：`http://192.168.1.1:12345`

---

## 插件设置

1. **云人升代理地址（推荐）**
  - 直接填完整地址：`http://你的IP:端口` 或 `https://你的IP:端口`
2. **任务清单 ID**
  - 填你希望写入的任务清单 ID
3. **日记同步模式**
  - 推荐 `手动（按钮/命令）`，避免把打字过程同步出去
4. **日记文件夹**
  - 例如：`Daily` 或 `journal/daily`
5. **日记标题格式（正则）**
  - 日期日记常用：`^\d{4}-\d{2}-\d{2}$`
6. **仅同步本地今天日记（可选）**
  - 打开后只同步当天文件名（如 `2026-03-02.md`）
7. 其他可选项
  - 默认任务金币奖励
  - 面板查询物品 ID
  - 启用属性查询（query_skill）

## 日常使用

### 手动同步（推荐）

1. 写完日记
2. 点击左侧功能区按钮“同步当前日记待办到 LifeUp”
  - 或执行命令：`同步当前日记待办到 LifeUp`

### 右键快速同步

- 在编辑器右键待办行：同步当前待办
- 选中多行后右键：批量同步选中待办

## 子任务规则（务必看）

Markdown：

- `- [ ] 喝水`
  - `- [x] 1`
  - `- [ ] 2`

同步后：

1. 创建主任务：`喝水`
2. 创建子任务：`1`、`2`
3. 已勾选子任务会自动标记完成

多级缩进会拍平为一层并拼接路径，例如：`二级 > 三级 > 四级`。

> 注意：LifeUp 本身只有一层 subtask。深层结构会变长、变乱，建议谨慎使用。

## 常见问题

### 单条能同步，整篇日记不动

通常是“日记匹配条件”未满足：

- 文件不在“日记文件夹”下
- 文件名不符合“标题正则”
- 开了“仅同步本地今天”，但不是今天文件

现在命令会弹出具体原因提示，按提示改即可。

### 自动模式把打字过程同步了

把“日记同步模式”改成 `手动`。
