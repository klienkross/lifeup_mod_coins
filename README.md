# lifeup_mod_coins

Obsidian 插件（草稿版）：在 Obsidian 日记与 LifeUp 之间同步待办（通过云人升 API）。

## 功能

- Obsidian -> LifeUp
- 右键同步当前待办 / 选中待办
- 日记同步（手动推荐，也支持自动）
- 支持子任务写入（多级会拍平）
- LifeUp -> Obsidian
- 按 `categoryId` 拉取任务列表，自动写入日记固定区块
- 数据面板
- 金币、番茄、物品拥有数、`query_skill`(1~6)

## 手机端准备

1. 安装 LifeUp：<https://lifeupapp.fun/zh/index.html#cta>
2. 在 LifeUp 开启开发者模式：侧边栏 -> 设置 -> 实验 -> 开发者模式
3. 安装并启动《云人升》
4. 获取服务地址（`IP:端口`），例如：`http://192.168.1.1:12345`

Google Play（云人升）：
<https://play.google.com/store/apps/details?id=net.lifeupapp.lifeup.http>

## 插件设置（推荐顺序）

### 1) 连接配置

- `云人升代理地址（推荐）`
- 建议直接填完整地址：`http://你的IP:端口` 或 `https://你的IP:端口`
- `请求超时（毫秒）`
- 默认 `10000`
- `LifeUp 代理主机/端口`
- 兼容旧配置，仅当“云人升代理地址”为空时生效

### 2) Obsidian -> LifeUp（日记同步）

- `任务清单 ID（category, type=task）`
- `默认任务金币奖励`
- `日记同步模式`
- 推荐 `手动（按钮/命令）`，避免把打字中间态同步出去
- `日记文件夹`，例如 `Daily`
- `日记标题格式（正则）`，例如 `^\d{4}-\d{2}-\d{2}$`
- `仅同步本地今天日记`（可选）

### 3) LifeUp -> Obsidian（写回日记）

- `启用从 LifeUp 写入日记`
- `写入触发模式`
- `自动（打开日记时）` 或 `手动（按钮/命令）`
- `拉取任务清单 ID（categoryId）`
- `写入区块标题`

### 4) 数据面板

- `启用属性查询（query_skill）`
- `面板查询物品 ID`

## 日常使用

### A. 写完日记后推送到 LifeUp（推荐）

1. 使用命令：`同步当前日记待办到 LifeUp`
2. 或点击侧边按钮：`同步当前日记待办到 LifeUp`

### B. 把 LifeUp 某个清单写回日记

1. 设置好 `拉取任务清单 ID（categoryId）`
2. 使用命令：`从 LifeUp 写入当前日记任务`
3. 或点击侧边按钮：`从 LifeUp 写入当前日记`

## 子任务规则

Markdown 示例：

- `- [ ] 喝水`
- `  - [x] 1`
- `  - [ ] 2`

同步到 LifeUp 时：

- 主任务会创建为任务
- 子项会创建为 subtask
- 已勾选子项会自动标记完成

多级子任务会拍平为一层并拼接路径，例如：`二级 > 三级 > 四级`。

注意：LifeUp 本身只有一层 subtask。深层结构较多时会显得杂乱。

## 写回日记区块说明

LifeUp 写回使用受控区块，插件会覆盖更新该区块而不是反复追加：

- `<!-- LIFEUP_TASKS_START -->`
- `<!-- LIFEUP_TASKS_END -->`

该区块会被排除出“反向推送到 LifeUp”的解析，避免循环同步。

## 常见问题

### 单条能同步，整篇日记不动

通常是日记匹配条件不满足：

- 文件不在日记文件夹中
- 文件名不符合标题正则
- 开了“仅同步本地今天”，但当前不是今天文件

命令现在会弹出具体原因提示。

### 自动模式把打字过程同步了

把 `日记同步模式` 改成 `手动（按钮/命令）`。
